Code Quality & Security

📖 Overview

SonarQube is an open-source platform developed by SonarSource for continuous inspection of code quality. It performs automatic reviews with static analysis of code to detect bugs, code smells, and security vulnerabilities across 30+ programming languages including Java, C#, JavaScript, TypeScript, Python, Go, and more.


SonarQube integrates into CI/CD pipelines and provides developers with actionable feedback on every commit. It enforces Quality Gates — a set of threshold conditions that code must meet before it can be released — ensuring that only clean, secure code reaches production.


Why it matters: Poor code quality leads to technical debt, security breaches, and costly production failures. SonarQube catches these issues early in the development lifecycle, reducing remediation costs by up to 10x compared to fixing issues in production.

🏗 How It Works

SonarQube follows a client-server architecture. Developers run the SonarScanner in their CI/CD pipeline, which analyzes source code and sends results to the SonarQube Server for processing, storage, and visualization.

Components:

  • SonarQube Server — Web server, search server (Elasticsearch), and compute engine
  • Database — PostgreSQL (recommended), Oracle, or MS SQL Server for storing configuration and analysis results
  • SonarScanner — Client-side tool that runs analysis and sends data to the server
  • SonarLint — IDE plugin providing real-time feedback in VS Code, IntelliJ, Eclipse

Quick Start with Docker

# Pull and run SonarQube Community Edition
docker run -d --name sonarqube \
  -p 9000:9000 \
  sonarqube:community

# Access the dashboard at http://localhost:9000
# Default credentials: admin / admin

# Install SonarScanner (macOS)
brew install sonar-scanner

# Run a scan in your project directory
sonar-scanner \
  -Dsonar.projectKey=my-project \
  -Dsonar.sources=. \
  -Dsonar.host.url=http://localhost:9000 \
  -Dsonar.token=YOUR_TOKEN
